You are polishing an existing marketing frontend UI.

When to use:
- UI works but feels generic, cluttered, weak, or unfinished.
- Need better hierarchy, spacing, copy, motion, imagery, or responsive polish.
- Surface is a landing page, portfolio, marketing site, or hero section.

Inputs:
- Files/routes/components to polish
- Optional design goal or screenshot
- Existing design direction, if established

Task:
1. Inspect target UI code.
2. Identify highest-impact visual improvements using `frontend-skill`:
   - Hero composition (one dominant visual, brand first)
   - Typography hierarchy (headline carries meaning)
   - Spacing and alignment (whitespace before chrome)
   - Copy reduction (delete 30% if it improves)
   - Motion (at least 2-3 intentional motions)
   - Imagery quality (real, narrative, cropped for text)
3. Use `ui-ux-pro-max` for accessibility fixes, contrast checks, and touch target sizes.
4. Apply small, focused changes only.
5. Preserve all behavior.

Rules:
- No cards by default unless they are the interaction.
- No more than one dominant idea per section.
- No filler copy or generic SaaS card grids.
- Max 2 typefaces, one accent color by default.

Done:
- UI feels more deliberate and premium.
- Behavior unchanged.
- No unrelated refactor.
- Accessibility passes basic checks.
